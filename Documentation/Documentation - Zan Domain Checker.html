General

We would like to thank you for purchasing Zan Domain Checker – a plugin is designed for checking top level domain (TLD) availability from your WordPress site.

Installation

You can install the theme in two ways: through WordPress, or via FTP. The zandc.zip file is the installable WordPress plugin and what you need to use to get the plugin installed. There are two ways to install Zan Domain Checker, via WordPress or via FTP. Click here for the WordPress method video tutorial, and here for the FTP method video tutorial. But first, you need to download the Zan Domain Checker files from your ThemeForest account. Navigate to your downloads tab on ThemeForest and find Zan Domain Checker.

WordPress Installation

How To Install Zan Domaun Checker via WordPress

Step 1 – Navigate to Plugins.

Step 2 – Click Add New and click the Upload Plugin button.

Step 3 – Navigate the zandc.zip file on your computer and click “Install Now”

Step 4 – Activate the newly installed plugin then you done the plugin installation.

FTP Installation

Step 1 – Log into your hosting space via an FTP software like File Zilla.

Step 2 – Unzip the zandc.zip file and only use the extracted zandc plugin folder.

Step 3 – Upload the extracted zandc plugin folder into wp-content > plugins folder.

Step 4 – Activate the newly installed plugin by going to Plugins and clicking the Activate button.

Plugin Settings

Zan Domain Checker settings give you full control plugin and allow you to customize the domain checker form design without touching a single line of code. Located in WP Dashboard > Zan Domain Checker, these options are global options. Some options can be overridden by shortcode for Visual Composer.

General Tab

Manage list of TLD extensions, domain checker form style, result message texts, search placeholder text, submit button text, whois page… etc.

1 – List of supported TLD extensions, each extension is separated by a vertical stripe. Ex: com|org|net|us|jp. Empty list means all extensions are allowed.

2 – Maximum number of TLD extensions to check at the same time. Default: 5.

3 – Search input placeholder text. Default: Search domain

4 – Turn the search submit button on or off (show or not show). Value is “yes” or “no”.

5 – If show the search submit button is turn on (“yes” is chosen), you can change the search button text. Default: Search.

6 – Set the way you want to show domain whois information: disable, popup or in a custom page. Default is popup. If “Custom Page” is chosen, the whois information will be shown in a page that you specified. Note: whois page must contain shortcode:[zwhois] or [zdcwhois]
7 – The title of whois page or popup. Default: Whois record for {domain}. The {domain} will be replaced by domain name.

8 – Manage the result message for available domain. The {domain} will be replaced by domain name.

9 – Manage the result message for not available domain. The {domain} will be replaced by domain name.

10 – Manage the message in case of not found any whois server supported the TLD extension.

11 – Restore default options.

12 –Save all options.

Integration Tab

Zan Domain Checker supported integration with WooCommerce, WHMCS or custom link.

1 – Choose integration type: WooCommerce, WHMCS, Custom Link or Disable.

2 – Top level domain extensions integration with products. Each extension and product id is separated by a vertical stripe. {ext1}-{product_id1}|{ext2}-{product_id2}|{ext3}-{product_id3}. Ex: com-23|org-18|net-65|us|jp-674. Note: You need create simple products for each TLD extension first. This options only show when integration type is WooCommerce.

3 – Integration button text for WooCommerce integration. Default: Add to cart. Note: This options only show when integration type is WooCommerce.

4 – Integration link. Put the custom link you want integration. The {domain} will be replaced by domain name. Ex for WHMSC: http://biling.yourhosturl.com. Ex for custom link: http://zanthemes.net/?regdomain={domain}. Note: This options only show when integration type is WHMCS or Custom Link.

5 – Integration button text for WHMCS or custom link integration. Default: Order. Note: This options only show when integration type is WHMCS or Custom Link.

Google reCAPTCHA tab

Zan Domain Checker supported Google reCAPTCHA to protect you from spam and abuse. What is reCAPTCHA?

1 – Turn reCAPTCHA on/off.

2 – reCAPTCHA site key. Get your reCAPTCHA site key here: https://www.google.com/recaptcha/admin

3 – reCAPTCHA secret key. Get your reCAPTCHA secret key here: https://www.google.com/recaptcha/admin

Shortcode And Visual Composer

Zan Domain Checker using shortcodes to display form and whois information on the frontend. Here are the shortcodes:

  • Display domain checker form: [zandc]
  • Whois information: [zwhois] or [zdcwhois]. To change title: [zwhois title="Whois record for {domain}"] (the {domain} will be relaced with domain name). To remove pre tabs wrap the whois information: [zwhois pre="no"]

Supported Visual Composer with 4 different styles and unlimit color for text, placeholder, input background, button color…
